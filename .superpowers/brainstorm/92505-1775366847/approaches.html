Implementation Approach

Three ways to build the animated Payment Flow visual

A

React Three Fiber (Three.js) — recommended

Full 3D scene using the same stack as fluid-glass.tsx. Floating node meshes (Store, Gateway, Bank, Approval) connected by curved tube paths. Animated sphere particles travel along the paths as data pulses. Bloom post-processing for the gold glow. Consistent with codebase patterns.

Pros

  • Most visually impressive
  • Consistent with existing 3D stack
  • Bloom + depth make it hero-worthy

Cons

  • Heavier bundle (already paid by other pages)
B

CSS 3D + SVG Animations — lightweight

Nodes as styled divs with CSS perspective tilt, SVG lines connecting them, and stroke-dashoffset animation to create traveling pulse dots. Zero new dependencies. Lucide icons inside each node.

Pros

  • Zero extra dependencies
  • Fast to implement
  • Very reliable

Cons

  • Less depth and polish than 3D
  • Harder to achieve the same premium look
C

React Three Fiber — Hub & Spoke

Same Three.js stack but with a hub-and-spoke topology: multiple payment methods (Visa, TNG, GrabPay, DuitNow…) feeding into a central Gateway node, which then fans out to Bank and Approval. Shows the many-to-one-to-many nature of a gateway.

Pros

  • Communicates ecosystem breadth
  • Visually rich

Cons

  • More complex to read at a glance
  • Harder to convey clear flow direction

I recommend A — it matches the codebase's existing 3D pattern and will look the most polished on the hero. Click to confirm or pick another.
