## Logo Integration Guidelines

### Format Requirements
- **Format:** SVG preferred (vector, infinitely scalable)
- **Fallback:** High-resolution PNG (300+ DPI for print quality)
- **Size:** 60-80px in the orbiting scene (scale via CSS/Three.js material)

### Color & Branding
- **Full-color logos:** Use official brand colors whenever possible
- **Monochrome alternatives:** Not needed for this scene (uses full color)
- **Aspect ratio:** Maintain original aspect ratio; do not distort

### Usage Rights & Attribution
- **Logos are trademarked:** Ensure compliance with each payment provider's branding guidelines
- **Fair use:** These logos are displayed for identification/informational purposes on M1Pay's official website
- **Attribution:** Generally not required for brand logos on official merchant websites, but cite sources in documentation (as done above)

### Implementation Notes
- Store SVG files in `/public/logos/payment-methods/` directory
- Create sub-folder per logo if needed: `/public/logos/payment-methods/fpx/`, `/public/logos/payment-methods/visa/`, etc.
- Use lazy-loading or optimize SVG sizes to minimize initial load
- Consider sprite-sheet or icon font if concerned about HTTP requests

---

## Quality Checklist Before Implementation

- [ ] All 10 logos downloaded from verified sources
- [ ] Each logo is in SVG format (or PNG with high resolution)
- [ ] Logos match official/current branding (no deprecated versions)
- [ ] File sizes optimized (SVGs minified if needed)
- [ ] Aspect ratios preserved
- [ ] Colors accurate to official branding
- [ ] No distortion or artifacts visible at 60-80px size
- [ ] Rights/licensing confirmed for web use
